# Getting Started

This guide is for new workspace owners and managers. Follow it after you create a TrustLoop account or accept an invite.

## What to do first

### 1. Sign in and land on the dashboard

Use any sign-in method available to you:

- Email verification code
- Google
- GitHub
- SAML SSO if your workspace uses it
- Invite link if you were added by a teammate

After a successful sign-in, you should land on the dashboard.

### 2. Open the settings area

Use the main menu to review these pages:

- **AI & API Keys**
- **Workspace**
- **Team**
- **Billing**

You do not need to change everything at once. Start with the items that affect daily incident work.

## First-time workspace setup

### Add at least one AI provider key

Open **Settings -> AI & API Keys** and add one active provider key.

TrustLoop can work without AI assistance, but AI triage and AI-written customer updates will stay unavailable until a provider key is active.

### Confirm your workspace slug

Open **Settings -> Workspace** and check the public workspace slug.

The slug matters because it is used for:

- Your public status page URL
- Some workspace-specific sign-in flows such as SAML SSO
- External references when your team shares the public incident stream

### Decide whether to enable the public status page

If you want customer-facing updates, turn on the public status page in **Settings -> Workspace**.

### Invite the right people

Open **Settings -> Team** and invite the people who need access.

Recommended minimum setup:

- At least one additional owner or manager
- The incident responders who will own incidents day to day
- Phone numbers for anyone who should receive urgent `P1` escalation messages

### Review billing and usage limits

Open **Settings -> Billing** to confirm your current plan and usage allowances.

## Your first live test

Run a short internal test before you rely on the workspace during a real incident.

### Recommended test flow

1. Create a sample incident from the dashboard.
2. Assign an owner.
3. Run AI triage.
4. Draft a customer update.
5. Publish one visible status update if your status page is enabled.
6. Resolve the incident.
7. Export the incident if your team needs an audit copy.

## What success looks like

You are ready to use TrustLoop when all of the following are true:

- At least one AI key is active.
- Your workspace slug is set.
- Owners and responders have access.
- At least one alert source or intake path is ready.
- You have completed one practice incident end to end.

## If you are stuck

### I signed up but cannot get into the dashboard

Read [Sign In and Access](/docs/guides/identity-and-access).

### AI actions are missing or failing

Go to **Settings -> AI & API Keys** and check that:

- A provider key was saved
- The key test passes
- The key is still marked active

### The status page is not available

Go to **Settings -> Workspace** and confirm:

- A workspace slug is present
- Public status page is enabled
- You have already published a visible update if you expect content to appear

<Callout type="info" title="Start small">
  The fastest successful rollout is usually one workspace, one AI provider key, one alert source, and one internal incident drill. You can expand after that without overwhelming the team.
</Callout>
