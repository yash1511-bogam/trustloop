# Getting Started

This guide walks new workspace owners and administrators through the initial setup process after creating a TrustLoop account or accepting a team invite.

## Prerequisites

- A TrustLoop account (via registration or team invite)
- Access to at least one AI provider API key (OpenAI, Gemini, or Anthropic)

## Step 1: Sign In

Authenticate using any supported method:

- Email verification code
- Google OAuth
- GitHub OAuth
- SAML SSO (if configured by your organization)
- Team invite link

After successful authentication, you will be directed to the dashboard.

## Step 2: Review Workspace Settings

Navigate to the settings area and review the following pages:

- **AI & API Keys** — Provider key configuration and workflow routing
- **Workspace** — Slug, status page, integrations, and SAML configuration
- **Team** — Member management, roles, and contact information
- **Billing** — Plan tier, usage, and payment status

## Initial Configuration

### Configure an AI Provider Key

Navigate to **Settings → AI & API Keys** and add at least one active provider key.

TrustLoop operates without AI assistance, but the following features require an active provider key:

- AI-powered incident triage
- Customer update draft generation

### Set the Workspace Slug

Navigate to **Settings → Workspace** and confirm the public workspace slug. The slug is used for:

- Public status page URLs
- SAML SSO authentication flows
- External workspace references

### Enable the Public Status Page

If your team publishes customer-facing incident updates, enable the public status page in **Settings → Workspace**.

### Invite Team Members

Navigate to **Settings → Team** and add the appropriate users:

- At least one additional owner or manager for redundancy
- Incident responders who will own and manage incidents
- Phone numbers for users who should receive P1 SMS escalations

### Review Billing and Quotas

Navigate to **Settings → Billing** to confirm your current plan tier and usage allowances.

## Validate Your Environment

Run a complete test cycle before relying on the workspace for production incidents.

### Test Procedure

1. Create a sample incident from the dashboard.
2. Assign an owner.
3. Run AI triage.
4. Generate a customer update draft.
5. Publish a visible status update (if the status page is enabled).
6. Resolve the incident.
7. Export the incident record (if your team requires audit copies).

## Readiness Checklist

Your workspace is production-ready when:

- At least one AI provider key is active and verified.
- The workspace slug is configured.
- Owners and responders have been granted access.
- At least one alert source or integration is connected.
- A test incident has been completed end to end.

## Troubleshooting

### Cannot access the dashboard after sign-up

See [Sign In and Access](/docs/guides/identity-and-access) for authentication troubleshooting.

### AI actions are unavailable or failing

Navigate to **Settings → AI & API Keys** and verify:

- A provider key has been saved.
- The key verification test passes.
- The key is marked as active.

### Status page is not accessible

Navigate to **Settings → Workspace** and confirm:

- A workspace slug is configured.
- The public status page is enabled.
- At least one visible status update has been published.

<Callout type="info" title="Recommended approach">
  Start with a single workspace, one AI provider key, one alert source, and one internal test incident. Expand your configuration incrementally after validating the core workflow.
</Callout>
