# Getting Started

This guide gets an operations lead from zero to first incident run in one session.

## 1. Create and verify your workspace

1. Open [Create workspace](/register).
2. Complete email verification.
3. Set workspace name and public status slug in **Settings -> Workspace**.

## 2. Configure incident operations defaults

1. Open **Settings -> Workspace**.
2. Enable status page if your team publishes customer-facing updates.
3. Confirm severity vocabulary (P1/P2/P3) and owner expectations with your support team.

## 3. Add AI provider keys (optional but recommended)

1. Open **Settings -> AI**.
2. Add at least one provider key (OpenAI, Gemini, or Anthropic).
3. Use the built-in key test action before enabling automated triage.

If no key is configured, TrustLoop still supports manual incident operation with workflow reminders.

## 4. Connect external alert sources

TrustLoop supports webhook-driven incident creation from tools like Sentry, Datadog, PagerDuty, and generic webhooks. Start with one source first, then scale.

## 5. Run a first incident drill

1. Create one test incident from your dashboard.
2. Assign an owner and publish one internal status update.
3. Confirm reminders, timeline events, and dashboard trend refresh.

## 6. Validate billing and lifecycle controls

1. Open **Settings -> Billing**.
2. Run checkout in test mode.
3. Confirm webhook processing, receipt email behavior, and grace-period handling.
