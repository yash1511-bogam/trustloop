# Documentation Style and Citations

## Writing standard used in TrustLoop docs

1. Write for the operator first: direct, explicit, and action-oriented.
2. Put key intent in headings and first lines.
3. Keep one job per page when possible.
4. Use consistent terms for the same concept across pages.

This standard aligns with major maintained guides that emphasize clarity, consistency, and reader-first structure<sup id="ref-google-about"><a href="#src-google-about">[1]</a></sup><sup id="ref-microsoft-welcome"><a href="#src-microsoft-welcome">[2]</a></sup>.

## Citation format

TrustLoop uses **numbered citations** in text and a **Sources** section at the bottom of the page:

- In-text: `[1]`, `[2]`, `[3]`
- Source entries include title, URL, retrieval date, and a backlink arrow (`↩`) to the citation callout.

This follows current style-guide direction favoring numeric footnotes and predictable formatting<sup id="ref-google-whatsnew"><a href="#src-google-whatsnew">[3]</a></sup>.

## Cross-links and external links

- Prefer adding short context on the current page before linking out.
- Use descriptive link text instead of vague text like "click here."
- Keep links selective to reduce decision overload for readers.
- Add internal backlinks so users can return to where they came from.

These rules follow established cross-reference guidance from Google and Microsoft documentation style guidance<sup id="ref-google-cross"><a href="#src-google-cross">[4]</a></sup><sup id="ref-microsoft-search"><a href="#src-microsoft-search">[5]</a></sup>.

## Information architecture rule

TrustLoop docs separate learning, task execution, explanation, and reference content. This keeps navigation predictable as the docs set scales<sup id="ref-diataxis"><a href="#src-diataxis">[6]</a></sup>.
