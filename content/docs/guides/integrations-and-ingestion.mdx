# Integrations

TrustLoop supports multiple integration methods for automated incident intake. Configure one integration first, validate it with a test event, then expand as needed.

## Integration Methods

### Webhooks

Connect monitoring and observability tools that support outbound HTTP webhooks.

Supported sources:

- Datadog
- PagerDuty
- Sentry
- Langfuse
- Helicone
- Generic JSON

### Workspace API Keys

Use API keys for custom scripts, internal services, or automation pipelines that create or update incidents programmatically.

### Slack

Enable incident creation, triage alerts, and customer update threads directly within Slack.

## Webhook Configuration

1. Navigate to **Settings → Workspace**.
2. Open the integrations section.
3. Select the source to connect.
4. Save or rotate the webhook secret.
5. Copy the endpoint URL and required headers into your external tool.
6. Send a test event to validate the connection.

### Example: Generic Webhook

<CodeBlock title="Send a signed generic webhook event">
  <Pre>
    <code className="language-bash">{`curl -X POST https://your-trustloop-app.com/api/webhooks/generic \\
  -H 'Content-Type: application/json' \\

## API Key Usage

Generate workspace API keys in **Settings → AI & API Keys**.

Use API keys when:

- Creating incidents from internal systems
- Updating incidents via trusted automation
- Authenticating without a session cookie

<CodeBlock title="Authorization header">
  <Pre>
    <code className="language-http">{`Authorization: Bearer sk-tl-xxxxxxxx.your-secret-value`}</code>
  </Pre>
</CodeBlock>

## Slack Configuration

1. Navigate to **Settings → Workspace**.
2. Connect the Slack application.
3. Save the Slack incident channel ID.
4. Create a test incident from Slack or from the TrustLoop dashboard.

Once connected, Slack supports:

- Incident creation via slash commands
- Triage alert delivery to the configured channel
- Customer update drafts within existing Slack threads

## Troubleshooting

### Webhook returns unauthorized

Verify the following:

- The workspace slug or ID in the request headers
- The webhook secret saved in TrustLoop
- The signature format and timestamp window

### Webhook created a duplicate warning

The payload included an external reference matching an existing incident. TrustLoop uses external references for deduplication.

### API key stopped working

Navigate to **Settings → AI & API Keys** and confirm the key is still marked as active.

### Slack connected but alerts are not posting

Navigate to **Settings → Workspace** and verify the Slack incident channel ID is saved.

<Callout type="info" title="Start with one integration">
  The recommended approach is one working alert source, one successful test incident, and one confirmed owner notification. Add additional integrations after validating the initial setup.
</Callout>
