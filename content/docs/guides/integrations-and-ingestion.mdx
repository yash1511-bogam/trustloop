# Integrations

TrustLoop supports multiple integration methods for automated incident intake. Configure one integration first, validate it with a test event, then expand as needed.

## Integration Methods

### Webhooks

Connect monitoring and observability tools that support outbound HTTP webhooks.

Supported sources:

- Datadog
- PagerDuty
- Sentry
- Langfuse
- Helicone
- Generic JSON

### Workspace API Keys

Use API keys for custom scripts, internal services, or automation pipelines that create or update incidents programmatically.

### Slack

Enable incident creation, triage alerts, and customer update threads directly within Slack.

## Webhook Configuration

1. Navigate to **Settings → Workspace**.
2. Open the integrations section.
3. Select the source to connect.
4. Save or rotate the webhook secret.
5. Copy the endpoint URL and required headers into your external tool.
6. Send a test event to validate the connection.

## API Key Usage

Generate workspace API keys in **Settings → AI & API Keys**.

When you create a key, TrustLoop now asks for two things in addition to the name:

1. A **usage preset** that preselects the right scopes for the job
2. An **expiry window** so the key automatically ages out if you forget to rotate it

Use API keys when:

- Creating incidents from internal systems
- Updating incidents via trusted automation
- Running triage or publishing updates from approved services
- Authenticating webhook ingestion without a browser session

### Recommended Usage Presets

| Preset | What it enables | Default scopes |
| --- | --- | --- |
| Read-only | Dashboards, exports, and systems that only inspect incidents | `incidents:read` |
| Incident intake | Creating and updating incidents from automation | `incidents:read`, `incidents:write` |
| Incident response | Running triage and publishing customer/status updates | `incidents:read`, `incidents:write`, `incidents:triage`, `customer-updates:write`, `status-updates:write` |
| Webhook intake | Authenticating calls to TrustLoop webhook endpoints with a Bearer key | `webhooks:ingest` |

You can keep the preset scopes as-is or fine-tune the individual permissions before saving the key.

### Create an API Key

1. Navigate to **Settings → AI & API Keys**.
2. In **Workspace API keys**, enter a descriptive name.
3. Choose the closest **usage preset**.
4. Review the preselected scopes and uncheck anything the integration does not need.
5. Choose an **expiry** such as 30 days, 90 days, or never.
6. Complete the Turnstile check and create the key.
7. Copy the revealed secret immediately. TrustLoop never shows the full secret again.

<CodeBlock title="Authorization header">
  <Pre>
    <code className="language-http">{`Authorization: Bearer sk-tl-xxxxxxxx.your-secret-value`}</code>
  </Pre>
</CodeBlock>

### Expiry and Rotation

- Expired keys are rejected automatically during authentication.
- Revoked keys remain visible in settings for auditability but can no longer be used.
- Prefer short-lived keys for external partners and long-running agents that can rotate automatically.
- If a key is shared with a third-party system, create a dedicated key instead of reusing one across integrations.

## Slack Configuration

1. Navigate to **Settings → Workspace**.
2. Connect the Slack application.
3. Save the Slack incident channel ID.
4. Create a test incident from Slack or from the TrustLoop dashboard.

Once connected, Slack supports:

- Incident creation via slash commands
- Triage alert delivery to the configured channel
- Customer update drafts within existing Slack threads

## Troubleshooting

### Webhook returns unauthorized

Verify the following:

- The workspace slug or ID in the request headers
- The webhook secret saved in TrustLoop
- The signature format and timestamp window

### Webhook created a duplicate warning

The payload included an external reference matching an existing incident. TrustLoop uses external references for deduplication.

### API key stopped working

Navigate to **Settings → AI & API Keys** and confirm the key is still marked as active and has not passed its expiry date.

### Slack connected but alerts are not posting

Navigate to **Settings → Workspace** and verify the Slack incident channel ID is saved.
