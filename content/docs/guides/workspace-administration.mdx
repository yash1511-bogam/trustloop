# Workspace Administration

TrustLoop organizes workspace controls into focused settings pages. This guide covers each configuration area and the settings most teams should prioritize.

## Settings Overview

### AI & API Keys

- Add, replace, or remove AI provider keys
- Test key connectivity and health
- Configure provider routing per AI workflow
- Generate workspace API keys for automation

### Workspace

- Set or update the public workspace slug
- Enable or disable the public status page
- Connect Slack
- Configure SAML SSO
- Manage webhook integration secrets
- Adjust workspace quotas
- Enable browser push notifications

### Team

- Invite team members
- Assign and modify roles (Owner, Manager, Agent)
- Remove members
- Add phone numbers for P1 SMS escalation
- Maintain user profile information

### Billing

- Review current plan tier and usage
- Apply coupon codes
- Initiate or manage checkout
- Monitor payment status

## Priority Configuration

### Workspace Slug

The workspace slug is required for:

- Public status page URLs
- SAML SSO authentication flows
- External workspace references

### AI Provider Keys

Without an active AI provider key:

- AI triage is unavailable
- Customer update draft generation is unavailable

### Team Roles

Recommended minimum configuration:

- At least two owners or managers for operational continuity
- Designated responders with incident ownership responsibilities
- Phone numbers for all users eligible for P1 SMS escalation

### Public Status Page

Enable the status page only when your team actively publishes customer-facing updates. An enabled status page with no published updates may appear incomplete to external visitors.

## Notification Channels

TrustLoop supports multiple notification methods for incident alerting and escalation.

### Email Reminders

Incident owners receive reminder emails for open incidents based on workspace configuration and severity-based intervals.

### Browser Push Notifications

Enable push notifications for real-time alerts and test notifications directly in the browser.

### SMS Escalation

P1 incidents can trigger SMS alerts when:

- The assigned owner or on-call responders have phone numbers configured
- The workspace is configured for urgent escalation

## Troubleshooting

### Workspace slug will not save

Verify the slug is lowercase, contains no special characters, and is not already in use by another workspace.

### Cannot remove or demote an owner

TrustLoop requires at least one owner per workspace. Promote another user to owner before removing or demoting the current one.

### Push notifications are not arriving

Verify the following:

- Browser notification permissions are granted
- Push notifications are enabled for the current browser session
- A test notification was sent successfully from workspace settings

### SAML configuration is not completing

Verify the workspace slug and IdP metadata configuration. If the identity provider is configured correctly but users cannot access the workspace, confirm they have been invited.

<Callout type="warn" title="Compliance mode is irreversible">
  Once compliance mode is enabled, it cannot be disabled. Compliance mode prevents incident deletion to maintain audit integrity.
</Callout>
