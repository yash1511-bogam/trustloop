# Navigation and Backlinks

## Sidebar behavior

The `/docs` section ships with a collapsible sidebar on desktop and a drawer menu on mobile. The sidebar starts expanded for top-level groups and supports manual collapse for focused reading.

## Link behavior in this docs set

- Internal links open in the same tab.
- External links use explicit destination text.
- Each policy page includes "Back to docs home" at top and in table-of-contents footer.
- Citation blocks include backlink arrows (`↩`) to the in-text citation markers.

This reflects mainstream technical documentation guidance for selective linking, descriptive link text, and reducing navigation friction<sup id="ref-google-cross-nav"><a href="#src-google-cross-nav">[1]</a></sup><sup id="ref-microsoft-search-nav"><a href="#src-microsoft-search-nav">[2]</a></sup>.

## Required backlink pattern

For any page containing external references:

1. Add citation markers in-line.
2. Add a numbered **Sources** section.
3. Add a backlink from each source line to its citation marker.
4. Add at least one internal route backlink (`/docs` or parent guide).

## Implementation note

The docs route uses Fumadocs `DocsLayout` and `DocsPage`, with:

- `sidebar.collapsible: true`
- `defaultOpenLevel: 1`
- table-of-contents enabled
- breadcrumb and page footer navigation enabled
