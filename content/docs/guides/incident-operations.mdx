# Incident Operations

This guide covers the end-to-end incident lifecycle within TrustLoop, from creation through resolution.

## Incident Lifecycle

### 1. Create an Incident

Incidents can be created from:

- The dashboard form
- Webhook integrations (Datadog, PagerDuty, Sentry, Langfuse, Helicone, Generic)
- Slack
- API-based automation

Provide a clear title describing the failure and a description covering impact, urgency, and customer risk.

### 2. Assign an Owner

Every incident should have a single, accountable owner assigned as early as possible. When an owner is assigned:

- The assignment is recorded in the incident timeline.
- The owner becomes eligible for follow-up reminder emails.
- P1 escalation can use the owner's contact details for urgent alerts.

### 3. Set Severity and Status

**Severity** indicates urgency:

| Level | Usage |
| --- | --- |
| `P1` | Urgent, customer-critical failures |
| `P2` | Significant impact with narrower scope |
| `P3` | Lower-risk incidents requiring tracking |

**Status** indicates progress:

| Status | Description |
| --- | --- |
| `NEW` | Incident created, not yet investigated |
| `INVESTIGATING` | Active investigation in progress |
| `MITIGATED` | Impact reduced, root cause may still be open |
| `RESOLVED` | Incident closed |

### 4. Run AI Triage

When an AI provider key is active, triage performs the following:

- Classifies severity and category
- Generates a concise summary
- Suggests next steps

For P1 and P2 incidents, reminder automation is queued automatically after triage.

### 5. Draft a Customer Update

Use the customer update action to generate external communication. TrustLoop uses the incident summary, current status, and recent timeline to draft the message.

### 6. Publish a Status Update

If the public status page is enabled, publish a visible update when customers need to see progress on the incident.

### 7. Resolve the Incident

Mark the incident as resolved to:

- Update the active incident queue
- Stop reminder automation
- Ensure accurate reporting and export data

## Dashboard Capabilities

The dashboard is designed for day-to-day incident operations:

- Filter by status, severity, category, and owner
- Search by title, description, or ticket reference
- Access incident details directly
- Export incident history
- Refresh reporting snapshots for leadership views

## Executive View

The executive page provides a high-level operational overview:

- Open incident volume and pressure
- P1 incident trends
- Created vs. resolved incident ratios
- Triage and customer update coverage
- Reminder delivery metrics

## Troubleshooting

### AI triage is unavailable

Navigate to **Settings → AI & API Keys** and verify:

- A provider key exists for the configured workflow
- The key is marked as active
- The key verification test passes

### Customer update drafting is unavailable

Verify AI key configuration on the same page. Customer updates use the configured workflow route and fall back to the triage route if no dedicated route is set.

### Published update not visible on the status page

Verify the following:

- The public status page is enabled in **Settings → Workspace**
- The workspace slug is configured
- The update was published with visibility enabled

### Cannot delete an incident

Compliance mode may be enabled for the workspace. When active, TrustLoop prevents incident deletion to maintain audit integrity.
