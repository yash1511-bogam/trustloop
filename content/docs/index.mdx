# TrustLoop Documentation

Use these guides after you create an account, accept an invite, or sign in for the first time. The goal is simple: help you get your workspace ready, understand how TrustLoop works, and quickly unblock common issues.

## Start here

| Guide | When to read it |
| --- | --- |
| [Getting Started](/docs/guides/getting-started) | You just created a workspace and want to finish setup. |
| [Sign In and Access](/docs/guides/identity-and-access) | You need help with email codes, Google, GitHub, SAML SSO, invites, or recovery. |
| [Run an Incident](/docs/guides/incident-operations) | You want to create, triage, update, publish, or close incidents. |
| [Connect Integrations](/docs/guides/integrations-and-ingestion) | You are connecting webhooks, API keys, or Slack. |
| [Configure Your Workspace](/docs/guides/workspace-administration) | You need to manage settings, AI keys, team access, status pages, or notifications. |
| [Plans, Billing, and Alerts](/docs/guides/billing-and-automation) | You need help with plan limits, payment issues, reminders, or escalation behavior. |

## Recommended setup order

1. Finish the steps in [Getting Started](/docs/guides/getting-started).
2. If anyone on your team is blocked from entering the app, use [Sign In and Access](/docs/guides/identity-and-access).
3. Connect one alert source with [Connect Integrations](/docs/guides/integrations-and-ingestion).
4. Create a test incident and follow [Run an Incident](/docs/guides/incident-operations).
5. Review [Configure Your Workspace](/docs/guides/workspace-administration) before inviting the rest of the team.
6. Check [Plans, Billing, and Alerts](/docs/guides/billing-and-automation) so you know how reminders, limits, and payment recovery work.

## What TrustLoop helps you do

- Receive incidents from your team, monitoring tools, webhooks, or Slack.
- Assign ownership and keep a single timeline of actions.
- Run AI triage and draft customer-safe updates.
- Publish visible status updates when customers need to be informed.
- Manage workspace settings, roles, API keys, notifications, and billing in one place.

<Callout type="info" title="Focused on real product behavior">
  These docs are written for people using the product. They explain what you can do in the TrustLoop UI, what the important setup steps are, and what to check first when something does not work as expected.

## If you are stuck

Start with the guide that matches the screen or task you are on. Each page includes a short troubleshooting section so you can usually fix the issue without leaving the app.
